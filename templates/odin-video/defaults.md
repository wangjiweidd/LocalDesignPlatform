# Odin Video Defaults

Use this file to avoid turning the Odin video line into a generic spec questionnaire.

These defaults are assumed unless the user clearly overrides them or a missing fact blocks accuracy.

## What Is Already Stable

- Account: Odin
- Domain: design job search, portfolio, interview, AI design workflow, designer productivity
- Audience: Chinese-speaking designers, especially job seekers and early-to-mid career product/UI designers
- Core promise: help designers make better judgment calls, not just learn tools
- Default renderer: HyperFrames HTML preview first
- Default output stop: `work/html/index.html` and `preview/index.html`
- Default canvas: `1440x1920`, `3:4`
- Default content shape: judgment -> evidence -> consequence -> action
- Default material priority: real screenshots, copied text, portfolio snippets, hiring evidence, user-provided references

## Default Odin Taste

- The video should feel like a designer's evidence board, not a SaaS dashboard and not a creator-template montage.
- The dominant mode should be one of:
  - `evidence-board`
  - `material-desk`
  - `keyword-stack`
  - `screen-focus`
- Titles should read as editorial judgments, not motivational slogans.
- Bottom captions should match the voiceover wording exactly.
- Motion should reveal reasoning order, not decorate idle space.
- Sound effects should be sparse and only tied to visible actions.

## Default Script Rules

- Open with a judgment or costly mistake, not a broad intro.
- Each paragraph should carry one clear point.
- Each key point should have visible evidence, material, or an annotation target.
- Avoid abstract career advice that could apply to any industry.
- Avoid writing the script like a slide lecture or social-media鸡汤.

## What Does Not Need Repeated Confirmation

Do not stop the flow to re-ask these unless the user has said something contradictory:

- whether the video is for designers
- whether Odin should sound like a design-career coach
- whether HTML preview comes before MP4
- whether the aesthetic should avoid generic SaaS cards
- whether evidence and annotations matter more than decorative motion
- whether voiceover and caption should stay semantically aligned

## What Still Needs Explicit Confirmation

Only ask when one of these is unknown and it would materially change the result:

- the exact topic judgment or contrarian point
- the target viewer segment if multiple are plausible
- the key evidence source when the script depends on it
- whether a claim is factual or still provisional
- whether a missing screenshot/material should be replaced with a placeholder
- whether the user wants a calm coaching tone or a sharper critique tone

## Fallback Rule

If the user brief is directionally clear but incomplete:

1. Keep the Odin defaults above.
2. Mark the gap as `[needs user confirmation]` only if it affects factual accuracy or the core judgment.
3. Otherwise proceed with the most conservative evidence-led version.

The Odin line should prefer momentum with strong defaults over excessive preflight questioning.
