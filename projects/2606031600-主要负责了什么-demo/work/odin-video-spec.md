# Odin Video Spec

## 1. Creative Brief

- Working title: 你在这个项目里主要负责了什么？
- Target viewer: 准备面试作品集项目讲解的视觉设计师
- Viewer pain: 说得很认真，但回答像执行清单，听不出判断力
- One-sentence takeaway: 这道题不是问你做了多少细节，而是看你能不能讲清设计范围、关键判断和结果证据
- Desired action after watching: 套用活动主视觉和品牌升级两段模板，改写自己的项目回答
- Platform and canvas: 3:4, 1440x1920
- Target length: 70 秒 HTML preview
- Evidence available: 原文口播脚本、两段可贴在视频中的完整回答模板
- Evidence missing: 真实项目截图，当前用文本证据板呈现

## 2. Odin Point Of View

- Surface topic: 面试高频题回答
- Deeper design-career problem: 视觉设计师容易把项目贡献讲成物料清单，面试官听不到设计判断
- Useful judgment: “主要负责了什么”不是细节题，而是逻辑题
- What this video refuses to become: 把口播全文做成字幕页；把模板内容挤成不可读的小字；只做静态轮播
- What Odin should say: 用一套清晰回答结构，把视觉设计师从执行者视角拉到决策者视角

## 3. Designer Aesthetic Direction

- Dominant visual mode: evidence-board + long answer document focus
- Visual taste: 像面试教练把一份完整话术模板贴到板上，边讲边框重点
- Typography attitude: 大标题是判断，正文模板是可读材料，红批注是教练提示
- Material texture: 淡绿色纸底、米色板、黑色标签、黄色高亮、红色批注
- Motion feel: 结构先落下，模板文档进入，红框依次圈出“范围 / 判断 / 结果”
- What must not appear: 轮播图片、SaaS 卡片、左侧装饰节点线、全文字幕页

## 6. Quality Gate

- On-screen template content must be readable and materially different from the bottom caption.
- Voiceover explains the structure; the interface shows the concrete answer.
- Each long template scene must show all five document blocks without clipping.
- Bottom caption keeps Xiaohongshu safe area.
- HTML preview is the default output. No MP4 export in this demo phase.
