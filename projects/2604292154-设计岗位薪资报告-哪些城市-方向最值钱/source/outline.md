# 小红书多图规划｜设计岗位薪资报告

## 假设

原始资料只有选题方向，没有提供招聘平台原始样本。本版按“公开招聘薪酬页面 + 求职判断框架”制作，不把任何薪资区间写成精确统计。后续如果补充岗位样本，可替换 P2-P5 的城市、方向和经验段判断。

## 角度锁定

- 目标读者：2-10 年经验，正在跳槽、换城市、换方向或谈薪的设计师。
- 真实亏点：只查平均薪资，容易低估自己，也容易因为城市、方向或岗位名做错选择。
- 隐藏标准：招聘方按城市预算、岗位职责、业务位置和项目证据定价，不按“设计师平均工资”定价。
- 可收藏交付：城市选择表、方向溢价判断、经验段证据、查薪避坑、跳槽复核清单。
- 触发类型：数字锚点、损失风险、隐藏规则、场景判断。

## 标题线程测试

P1：设计岗薪资，别只查平均数  
P2：同样 3 年，城市先拉开差距  
P3：一线高薪，不等于都该去  
P4：最值钱的方向，都靠近业务  
P5：3 年、5 年、10 年，涨价证据不同  
P6：高薪作品集，不是更会排版  
P7：查薪资，别被 4 个数字骗了  
P8：下一次跳槽，按这张表复核

## 页面规划

### P1 封面

- 页面角色：强钩子 + 对象识别。
- 触发类型：损失风险、隐藏规则。
- 主标题：设计岗薪资，别只查平均数。
- 副标题：城市、方向、年限，哪个最影响你的定价？
- 正文重点：平均数会把城市、行业、岗位级别和薪资结构混在一起。
- 可用单位：三变量判断公式。
- 视觉重点：多行大标题、三变量卡片、底部红色判断条。
- 布局类型：dense cover。
- HTML 落版：封面主视觉为超大黑字，右侧放变量标签，底部放一句判断。

### P2 城市问题唤醒

- 页面角色：解释同年限差距为什么出现。
- 触发类型：场景判断、损失风险。
- 主标题：同样 3 年，城市先拉开差距。
- 副标题：不是你不值钱，是岗位预算不一样。
- 正文重点：城市影响岗位密度、公司预算和跳槽机会。
- 可用单位：三类城市的适合人群。
- 视觉重点：高上限、均衡机会、成本友好三列卡片。
- 布局类型：comparison。
- HTML 落版：三列城市梯队 + 底部红条。

### P3 城市选择表

- 页面角色：把城市选择从排名改成决策。
- 触发类型：隐藏规则。
- 主标题：一线高薪，不等于都该去。
- 副标题：看岗位密度，也看生活成本。
- 正文重点：北京、上海、深圳、杭州机会不同，不能只为“高薪城市”迁移。
- 可用单位：城市迁移前 4 个问题 + 城市/机会/适合人群表。
- 视觉重点：城市判断表。
- 布局类型：dense。
- HTML 落版：上方问题卡，下方表格。

### P4 方向上限

- 页面角色：解释哪些方向更容易拿溢价。
- 触发类型：认知冲突、隐藏规则。
- 主标题：最值钱的方向，都靠近业务。
- 副标题：岗位名不重要，定价来自负责范围。
- 正文重点：产品设计、UX、交互、B 端、设计系统、品牌策略的溢价原因。
- 可用单位：方向溢价判断标准。
- 视觉重点：四层方向卡片 + 黑底结论。
- 布局类型：list + dense。
- HTML 落版：编号条卡，每条给判断标准。

### P5 经验段

- 页面角色：拆 3 年、5 年、10 年的定价证据。
- 触发类型：数字锚点、场景判断。
- 主标题：3 年、5 年、10 年，涨价证据不同。
- 副标题：年限只给入场券，结果决定上限。
- 正文重点：执行质量、负责范围、组织影响力。
- 可用单位：作品集需要补的证据。
- 视觉重点：三段经验卡 + 面试追问。
- 布局类型：flow。
- HTML 落版：3/5/10 年横向步骤。

### P6 作品集证据

- 页面角色：把薪资判断落到作品集修改。
- 触发类型：认知冲突、行动结果。
- 主标题：高薪作品集，不是更会排版。
- 副标题：它能证明你值得更高预算。
- 正文重点：低价信号 vs 高价信号；每个项目补目标、职责、决策、结果。
- 可用单位：作品集改写清单。
- 视觉重点：左右对照。
- 布局类型：comparison。
- HTML 落版：低价信号/高价信号双栏。

### P7 查薪避坑

- 页面角色：纠正常见误读。
- 触发类型：损失风险、数字锚点。
- 主标题：查薪资，别被 4 个数字骗了。
- 副标题：月薪、年包、范围、样本都要拆开看。
- 正文重点：月薪不等于年包，上限不等于你的报价，平均不等于岗位预算，岗位名不等于职责。
- 可用单位：20 个 JD 复核动作。
- 视觉重点：四个误区 + 建议动作。
- 布局类型：list。
- HTML 落版：四张误区卡，底部放复核动作。

### P8 行动清单

- 页面角色：保存用速查表。
- 触发类型：行动结果。
- 主标题：下一次跳槽，按这张表复核。
- 副标题：城市、方向、年限，最后都要变成求职动作。
- 正文重点：低预算城市、低上限方向、3-5 年卡点、8-10 年卡点分别怎么处理。
- 可用单位：跳槽前复核矩阵 + 投递/报价/作品集/换城市检查项。
- 视觉重点：四行矩阵 + 四格检查项 + 底部红条。
- 布局类型：dense。
- HTML 落版：四行矩阵，底部红条收口。
